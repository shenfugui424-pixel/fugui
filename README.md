# A股板块估值温度榜

每天早上 8 点自动刷新的网页：按"估值温度"从低到高列出 A 股各板块（越低越被低估），并给出每个板块对应的场外跟踪基金及其规模、费率、跟踪误差。

- 估值口径：板块当前 PE、PB 在近 10 年的历史百分位综合值（0–100），参考「且慢」「有知有行」做法。
- 数据来源：蛋卷基金（估值分位）、天天基金/东方财富（基金规模·费率·跟踪误差）。
- **免责声明：本页为个人研究工具，低估不等于一定上涨，不构成任何投资建议，据此操作风险自负。**

## 部署到网上（每天 08:00 自动刷新）—— 小白版步骤

> 整个过程在浏览器里点点点就行，不用写代码。

### 1. 注册并登录 GitHub
打开 https://github.com ，注册一个账号并登录（免费）。

### 2. 新建一个仓库（repository）
- 点右上角「+」→「New repository」。
- Repository name 填一个英文名，例如 `guzhi`。
- 选 **Public**（公开；GitHub Pages 免费版需要公开）。
- 点「Create repository」。

### 3. 上传本文件夹里的所有文件
- 在新仓库页面点「uploading an existing file」。
- 把本文件夹（`valuation-site`）里的 **所有文件和文件夹** 拖进去，包括：
  `fetch_data.py`、`build_site.py`、`requirements.txt`、`.gitignore`、
  `.github` 文件夹、`public` 文件夹。
  - ⚠️ `.github` 是隐藏文件夹但必须上传——它是自动刷新的核心。若拖拽时看不到它，
    可在 GitHub 网页点「Create new file」，文件名输入 `.github/workflows/daily.yml`，
    再把本地同名文件内容粘进去。
- 点「Commit changes」。

### 4. 打开「GitHub Actions」部署方式
- 进入仓库「Settings」→ 左侧「Pages」。
- 「Build and deployment」的「Source」选 **GitHub Actions**。

### 5. 手动跑一次，立刻出网址
- 进入仓库顶部「Actions」标签。
- 左侧选「daily-refresh」→ 右侧点「Run workflow」→ 绿色按钮「Run workflow」。
- 等 1–2 分钟跑完（出现绿色对勾）。
- 回到「Settings」→「Pages」，页面顶部会显示你的网址，形如：
  `https://你的用户名.github.io/guzhi/`
- 打开它就是你的网站。以后每天北京时间约 08:00 会自动刷新一次。

## 关于"国内能不能直接打开"

现在用的是 GitHub Pages（海外免费托管）。**国内访问可能偏慢或偶尔打不开**，先用来验证效果。
等你觉得内容满意了，可以告诉我，我帮你迁到国内稳定方案（任选其一）：

- **备案域名 + 阿里云/腾讯云**：国内访问最稳，是真正的独立网址；需要你有一个域名并完成备案（约 3–7 天）。
- **飞书在线表格**：每天自动把这张榜单写进一张飞书多维表格，国内手机秒开、零备案、零成本（你本来就常用飞书）。

## 常见问题

- **想改成几点刷新？** 编辑 `.github/workflows/daily.yml` 里的 `cron: "0 0 * * *"`。
  这里的时间是 UTC，比北京时间晚 8 小时（`0 0` = 北京 08:00；想要北京 09:00 就改成 `0 1`）。
- **想自己加/减板块？** 数据来自蛋卷的 A 股指数列表，脚本会自动过滤海外/港股/QDII。
  如需自定义板块范围，告诉我即可调整。
- **某个板块显示"场外基金待补充"？** 说明该指数暂时没找到合适的场外联接基金（多为小众指数）。
- **海外托管跑数据会不会失败？** GitHub 的服务器在境外，访问国内财经接口偶有不稳；
  若某天没刷新，可到 Actions 页面手动「Run workflow」重跑，或迁到国内方案。
